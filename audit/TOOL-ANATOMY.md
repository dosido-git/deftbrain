# DeftBrain Tool Anatomy — the canonical template

**This is the single source of truth for what every tool is made of.** It lists
every required component, its placement, its exact rendering parameters, the rule
that governs it, and whether a gate enforces it. Use it two ways:

- **New tool:** copy the skeleton (§7), fill each component per its spec below.
- **Existing tool:** compare part-by-part; every ✅-gated row is checked on push.

**Reference implementation:** `src/tools/MarkupDetective.js` + `backend/routes/markup-detective.js`.
When this doc and a tool disagree, the tool is wrong (unless it's a sanctioned
exception — §8). When this doc and prose in another audit file disagree, **this
doc wins** (see CONVENTIONS.md → Source-of-truth precedence).

Rule detail, scan commands, and rationale live in `CONVENTIONS.md` (PF-*/S7-*).
This doc is the *map*; CONVENTIONS.md is the *rulebook*. Gate column: ✅ = failed
push blocks; ⚠️ = convention, not gated (reviewer-checked).

## 1. The three layers (must stay in sync — CLAUDE.md rule 5)

| Layer | File | Purpose |
|---|---|---|
| Frontend component | `src/tools/<PascalCase>.js` | UI, inputs, results render, copy/print |
| Backend route | `backend/routes/<kebab-case>.js` | prompt, Claude call, JSON response (auto-mounted) |
| Catalog + i18n | `src/data/tools.js` entry + `src/i18n/locales/tools/<kebab>.js` | title/tagline/icon/category/color; all `t()` keys |

A change to any one requires checking the other two.

## 2. Frontend — import block (PF-1 / PF-18d)

or ActionButton imports fail PF-18a/PF-18d.

## 3. Frontend — the `c={}` color config (PF-2, PF-14, PF-15, PF-24)

Every tool derives `isDark` from `useTheme()` and builds a `c` object **before the
first `useState`** (PF-14 hook ordering). Never hardcode a light-only color
(CLAUDE.md rule 3). Destructure `const { isDark } = useTheme()` — `const { theme }`
fails PF-24. Canonical keys (copy from the reference; extend, don't rename):

```
card cardAlt input text textSecondary textMuted labelText accentTxt
btnPrimary btnSecondary border success warning danger successTxt warningTxt
pillActive pillInactive required
```

- `required: isDark ? 'text-amber-400' : 'text-amber-700'` — mandatory, used by every `*` (PF-15).
- `textMuted: isDark ? 'text-zinc-400' : 'text-gray-500'` — WCAG-AA floor (do not use zinc-500/gray-400).
- `label` is a **required alias**, not banned (CONVENTIONS Clarification 1).
- Banned color families & the `colorKey` indirection pattern: CONVENTIONS Clarifications 2–3.

## 4. Frontend — component render order

Everything sits inside the `ToolPageWrapper` frame. **Never** add a root
background/frame div (PF-11) — the wrapper provides it. Order top-to-bottom:

| # | Component | Placement | Rendering params (verbatim) | Rule | Gate |
|---|---|---|---|---|---|
| 1 | **Header card** | First `c.card`; MUST contain the inputs too (not a header-only card) | `<div className={\`${c.card} border ${c.border} rounded-xl shadow-sm\`}>` then a divider row `<div className="pb-3 border-b border-zinc-500 flex items-center justify-between">` | PF-3, PF-4 | ✅ |
| 2 | **Icon + title** | `<h2>` in the header, left | `<h2 className={\`text-xl font-bold ${c.text}\`}><span className="me-2">{tool?.icon ?? '🧩'}</span>{tool?.title ?? t('xx_title')}</h2>` — icon+title from `tools.js`, fallback via `t()` (never hardcoded — PF-4; fallbacks must match tools.js — Clarification 5) | PF-4 | ✅ |
| 3 | **Tagline** | `<p>` directly under the title | `<p className={\`text-sm ${c.textSecondary}\`}>{tool?.tagline ?? t('xx_tagline')}</p>` | PF-4 | ✅ |
| 4 | **Try-example pill** | **Own line, immediately after the tagline `<p>`** — nowhere else, no variants | `<button onClick={loadExample} disabled={loading} style={{ backgroundColor: (tool?.headerColor ?? '#888888') + '80' }} className={\`mt-1.5 px-2.5 py-0.5 rounded-full text-xs font-medium border disabled:opacity-40 ${isDark ? 'text-white border-white/40' : 'text-gray-800 border-transparent'}\`}>{t('try_example')}</button>` — required on 2+ input tools; `loadExample` is **no-arg** (multi-example → `EXAMPLES[0]`/active mode); never a chip picker | PF-17, PF-17b, PF-28 | ✅ |
| 5 | **Reset button** | Header **top-right**, on the title row; render with a **ternary** (not `&&`) gated on "any input or result present" | `<button onClick={handleReset} className={\`${c.btnSecondary} px-3 py-1.5 rounded-lg text-xs font-bold\`}>↺ {t('start_over')}</button>` — exactly **one**, `c.btnSecondary` (never `btnPrimary`), never inside a `{results && (…)}` block | PF-16, PF-16-EXC, PF-27, Clar. 4 | ✅ |
| 6 | **Input card** | Same card as header (folded in) or the next `c.card`; never a header-only card | `border ${c.border} rounded-xl shadow-sm` (no `shadow-lg`); inputs use `c.input` | PF-3 | ✅ |
| 7 | **Required-field `*`** | On the `<label>` of every required input | `<span className={c.required}>*</span>` — every asterisk goes through `c.required` | PF-15 | ✅ |
| 8 | **Submit button** | Below inputs | Full-width `c.btnPrimary`; loading state uses `tool?.icon` with `animate-spin` (not a hardcoded ⏳ — S0/PF-14); disabled buttons carry `disabled:opacity-40` (PF-13) | PF-13, PF-14 | ✅ |
| 9 | **Growable input lists** | If inputs are add-able | On add/Enter, focus follows to the new field | PF-19 | ✅ |
| 10 | **Error state** | Below submit | `error` string rendered in JSX; **never** render `error.message` raw (leaks internals — PF-20) | PF-20 | ✅ |
| 11 | **Results renderer** | Below the form (own cards) | Each result card `${c.card} border ${c.border} rounded-xl`; access nested fields with optional chaining `results?.x?.y` (PF-21); numbers the UI consumes are code-computed, not model-trusted | PF-21 | ✅ |
| 12 | **Results scroll** | When results land below the fold | `useEffect` scrolls to results; every `setTimeout` has a `clearTimeout` return | PF-7 | ⚠️ |
| 13 | **Copy / Print / Share** | Results action row | Shared `CopyBtn`/`PrintBtn`/`ShareBtn`/`ActionBar` from `../components/ActionButtons` — never inline copy logic, never a local `CopyBtn`, never `copiedField` state (PF-5, PF-22) | PF-5, PF-22 | ✅ |
| 14 | **`buildFullText` + ActionBar reg** | After result state, before render | `useRegisterActions(buildFullText(), tool?.title || 'Name')` **after** `buildFullText` (TDZ — PF-8/PF-26); `buildFullText` includes **every** result variable and appends `BRAND` (PF-8, PF-18b) | PF-8, PF-26, PF-18b | ✅ |
| 15 | **Cross-references** | Sidebar/footer | ≥1 `/PascalCase` link, emoji-prefixed, no `target="_blank"`, ≤3 per cluster; if `CROSS_REFS` const defined it must render (PF-18c) | PF-9, PF-18c | ✅ |
| 16 | **Keyboard handler** | `useEffect` | ⌘/Ctrl+Enter submits via `handleXxxRef.current?.()`; guards on `SELECT` only; every submit-capable view wired (Clar. 7) | PF-6 | ⚠️ |
| 17 | **History** (optional) | If the tool keeps a session list | `usePersistentState`; cap ≤ 6 (higher needs a documented exception — PF-25); preview ≈ 40 chars (Clar. 6); never name the state `history` (shadows router — PF-12b) | PF-25, PF-12b | ✅ |

## 5. Frontend — localization (4 layers, all required for a "localized" tool)

| Layer | What | Where |
|---|---|---|
| UI strings | every user-facing string is `t('key')`, key defined for all 13 langs | `src/i18n/locales/tools/<kebab>.js` |
| Value formatting | currency/number/date via `formatLocale.js` — never hardcode a `$` | JSX |
| Output language | backend `withLanguage(prompt, userLanguage)` | route |
| Locale reasoning | backend `withLocaleContext(userLocale, userCurrency, userRegion)` on `system:` | route |

Four-field locale context is locked — CONVENTIONS Clarification 11. `web_search`
tools need frontend `stripCitesDeep` — Clarification 10. RTL: use logical Tailwind
props (`ms-`/`me-`/`ps-`/`pe-`/`text-start`) — CONVENTIONS RTL section.

## 6. Backend — route anatomy (S7)

Auto-mounted: dropping a `.js` in `backend/routes/` makes it live (S7.8); never
`router.use(rateLimit(...))` (throttles the whole site — apply per-route).

```js
const express = require('express');
const router = express.Router();
const { withLanguage, withLocaleContext, callClaudeWithRetry } = require('../lib/claude');
const { MODELS } = require('../lib/models');
const { rateLimit } = require('../lib/rateLimiter');

router.post('/<kebab>', rateLimit(), async (req, res) => {
  try {
    const { /* fields */, userLanguage, userLocale, userCurrency, userRegion } = req.body;
    if (!/* required */) return res.status(400).json({ error: '…' });
    // input caps on any pasted/large field
    const prompt = withLanguage(`…schema…`, userLanguage) + withLocaleContext(userLocale, userCurrency, userRegion);
    const parsed = await callClaudeWithRetry({ model: MODELS.SMART, max_tokens: 4000, messages: [{ role:'user', content: prompt }] }, { label: '<kebab>' });
    if (!parsed.<top_level_always_present_key>) return res.status(500).json({ error: '…' });
    res.json(parsed);
  } catch (error) {
    console.error('[<Name>]', error);
    res.status(500).json({ error: 'Something went wrong. Please try again.' });
  }
});
module.exports = router;
```

| Rule | Requirement | Gate |
|---|---|---|
| S7.1 | import `withLanguage`; `cleanJsonResponse` where JSON.parse is used | ✅ |
| S7.4 | one `withLanguage()` per API call (name-keyed; each split call calls it itself) | ✅ |
| S7.5 | `web_search` output → `stripCites` | ✅ |
| S7.10 | `max_tokens` sized to the schema (bound it; ≥800 on structured JSON) | ✅ |
| S7.12 | `callClaudeWithRetry` calling convention (simple-string honors `options.system`) | ✅ |
| S7.13 | response-guard field must be a **top-level, always-present** schema key (nested/nullable → 500 every call) | ✅ |
| — | try/catch around the whole handler; generic client error, log server-side | ⚠️ |
| — | model IDs via `MODELS.*` roles, never hardcoded | ⚠️ |

Big/slow mega-schema → parallel-split (disjoint keys, per-call `withLanguage`,
ALL-N-KEYS rule). Model-generated HTML/SVG must be sanitized (DOMPurify + server
strip). See CONVENTIONS + SECURITY-REVIEW.

## 7. New-tool skeleton

1. `src/tools/<PascalCase>.js` — copy MarkupDetective; keep the import block (§2),
   `c` block (§3), and render order (§4).
2. `backend/routes/<kebab>.js` — copy the §6 skeleton.
3. `src/data/tools.js` — add `{ id, title, tagline, icon (emoji), category, headerColor, … }`.
4. `src/i18n/locales/tools/<kebab>.js` — `xx_title/xx_tagline/…` for all 13 langs; merge into `RESOURCES`.
5. Run the 5 gates (CLAUDE.md); add to `LOCALIZED_TOOLS` when fully localized; golden-lock.

## 8. Sanctioned exceptions (do not "fix" these)

- **Emergency tools** (SafeWalk, DriveHome, etc.) — may break UI conventions for safety-first layout (CONVENTIONS Clarification 9).
- **PF-16 multi-view** — a tool with N submit views may key reset per view (PF-16-EXC).
- **App-chrome English** — `tools.js` title/description are English by design; not part of localization.
